# Phase 5 — Demo Apps Deployment (CMS-Driven)

## Goal
Deploy demo apps to the VPS as Docker containers. All routing configuration (subdomain ↔ port mapping) is managed entirely from Payload CMS — no manual Nginx config needed per demo. Adding a new demo = add entry in CMS → run one sync script.

---

## How It Works

```
CMS (Demos collection)
  title: "RumahTAP"
  subdomain: "rumahtap"
  port: 3002
  is_active: true
        │
        ▼
  nginx-sync.sh   ← reads CMS API
        │
        ▼
  generates: nginx/conf.d/demo-rumahtap.conf
        │
        ▼
  nginx reload → rumahtap.yourdomain.com live ✅
```

## Demo Registry (Manage in CMS)

Go to `https://cms.yourdomain.com/admin` → **Demos** → Create New.

Fields to fill for each demo:

| Field | Example |
|---|---|
| Title | RumahTAP |

## Step 1 — nginx-sync Script

This script reads all active demos from the CMS API, generates an Nginx config file for each, and reloads Nginx. Run it on the VPS whenever you add or change a demo in CMS.

Create `/opt/hendrik-portfolio/scripts/nginx-sync.sh`:

```bash
#!/bin/bash
set -e

DOMAIN="yourdomain.com"
CMS_API="http://localhost:3001/api"
NGINX_CONF_DIR="/opt/hendrik-portfolio/nginx/conf.d"
CERTBOT_WWW="/var/www/certbot"

echo "🔄 Fetching demo list from CMS..."

# Install jq if not present
if ! command -v jq &> /dev/null; then
  apt-get install -y jq > /dev/null
fi

# Fetch all active demos from Payload CMS API
DEMOS=$(curl -s "${CMS_API}/demos?limit=100&where[is_active][equals]=true")
DEMO_COUNT=$(echo "$DEMOS" | jq '.docs | length')
echo "📦 Found ${DEMO_COUNT} active demo(s)"

# Remove old generated demo configs (keep portfolio.conf and cms.conf)
rm -f "${NGINX_CONF_DIR}"/demo-*.conf

# Generate a config file per demo
echo "$DEMOS" | jq -c '.docs[]' | while read -r demo; do
  SUBDOMAIN=$(echo "$demo" | jq -r '.subdomain')
  PORT=$(echo "$demo" | jq -r '.port')
  TITLE=$(echo "$demo" | jq -r '.title')
  CONF_FILE="${NGINX_CONF_DIR}/demo-${SUBDOMAIN}.conf"

  echo "  ✍️  ${SUBDOMAIN}.${DOMAIN} → port ${PORT}"

  cat > "$CONF_FILE" << EOF
# Auto-generated by nginx-sync.sh — DO NOT EDIT MANUALLY
# Demo: ${TITLE}
server {
  listen 80;
  server_name ${SUBDOMAIN}.${DOMAIN};

  location /.well-known/acme-challenge/ {
    root ${CERTBOT_WWW};
  }

  location / {
    return 301 https://${SUBDOMAIN}.${DOMAIN}\$request_uri;
  }
}

server {
  listen 443 ssl;
  server_name ${SUBDOMAIN}.${DOMAIN};

  ssl_certificate /etc/letsencrypt/live/${DOMAIN}/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/${DOMAIN}/privkey.pem;

  location / {
    proxy_pass http://host.docker.internal:${PORT};
    proxy_set_header Host \$host;
    proxy_set_header X-Real-IP \$remote_addr;
    proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto \$scheme;
  }
}
EOF
done

echo "🔁 Reloading Nginx..."
docker compose -f /opt/hendrik-portfolio/docker-compose.yml exec nginx nginx -s reload

echo "✅ Nginx sync complete!"
echo ""
echo "Active demo subdomains:"
echo "$DEMOS" | jq -r '.docs[] | "  https://\(.subdomain).'"${DOMAIN}"' → port \(.port)"'
```

```bash
chmod +x /opt/hendrik-portfolio/scripts/nginx-sync.sh
```

## Step 2 — Running the Sync

Whenever you add, edit, or remove a demo in CMS:

```bash
# Option A: SSH in and run directly
ssh root@YOUR_VPS_IP "bash /opt/hendrik-portfolio/scripts/nginx-sync.sh"

# Option B: via Makefile shortcut (from local machine)
make sync-demos
```

Add to `Makefile`:

```makefile
sync-demos:
	ssh root@YOUR_VPS_IP "bash /opt/hendrik-portfolio/scripts/nginx-sync.sh"
```

## Step 3 — Deploying a Demo App

Each demo lives in `/opt/hendrik-portfolio/demos/[subdomain]/` on the VPS.

### A. Upload the App

```bash
# Option 1: Clone from GitHub
ssh root@YOUR_VPS_IP
mkdir -p /opt/hendrik-portfolio/demos/rumahtap
git clone https://github.com/yourusername/rumahtap-demo.git /opt/hendrik-portfolio/demos/rumahtap

# Option 2: rsync from local machine
rsync -avz ./demos/rumahtap/ root@YOUR_VPS_IP:/opt/hendrik-portfolio/demos/rumahtap/
```

### B. Dockerfile per Demo

**Laravel:**

```dockerfile
FROM php:8.2-fpm-alpine
WORKDIR /var/www/html
RUN docker-php-ext-install pdo pdo_pgsql pdo_mysql
RUN curl -sS https://getcomposer.org/installer | php -- --install-dir=/usr/local/bin --filename=composer
COPY . .
RUN composer install --no-dev --optimize-autoloader
COPY .env.demo .env
RUN php artisan key:generate && php artisan migrate --seed --force
EXPOSE 8000
CMD ["php", "artisan", "serve", "--host=0.0.0.0", "--port=8000"]
```

**Node.js:**

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY . .
EXPOSE 3002
CMD ["node", "server.js"]
```

### C. Add Service to docker-compose.yml

```yaml
  demo-rumahtap:
    build: ./demos/rumahtap
    container_name: demo_rumahtap
    restart: unless-stopped
    env_file: ./demos/rumahtap/.env
    ports:
      - "127.0.0.1:3002:8000"   # host port = CMS port field, container port = app's internal port
    networks:
      - web
```

> **Rule:** Left port (host) must match the `port` value in CMS. Right port (container) is what the app listens on.

### D. Start the Container

```bash
cd /opt/hendrik-portfolio
docker compose up -d demo-rumahtap
docker compose logs demo-rumahtap -f
```

## Step 4 — Full Workflow: Adding a New Demo

```
1. Deploy app to /opt/hendrik-portfolio/demos/[name]/
2. Add Dockerfile + .env to the app
3. Add service to docker-compose.yml (127.0.0.1:PORT:INTERNAL_PORT)
4. docker compose up -d demo-[name]
5. CMS → Demos → Create new entry (fill subdomain + port + details)
6. make sync-demos
7. ✅ https://[name].yourdomain.com is live
```

---

## Step 5 — Disabling a Demo (Without Deleting)

1. CMS → Demos → uncheck `Is Active`
2. `make sync-demos`

Result:
- Nginx config removed → subdomain goes dark
- Portfolio `/demos` page hides the Launch button (or shows "Currently Unavailable")
- Container keeps running — re-enable anytime by rechecking Is Active + re-syncing

## Step 6 — Demo Data Best Practices

- [ ] Use seed data only — no real user/production data
- [ ] Set demo credentials in CMS `demo_credentials` field (shown on portfolio)
- [ ] Add banner in app UI: `⚠️ Demo Mode — Data resets every 24h`
- [ ] Disable external API calls (payment, OTP, etc.) in demo mode
- [ ] Read all config from `.env` — no hardcoded values

**Optional: Daily reset cron:**

```bash
# /opt/hendrik-portfolio/scripts/reset-demo-rumahtap.sh
#!/bin/bash
docker exec demo_rumahtap php artisan migrate:fresh --seed --force
echo "Reset: $(date)"
```

```bash
crontab -e
# Add:
0 2 * * * /opt/hendrik-portfolio/scripts/reset-demo-rumahtap.sh >> /var/log/demo-reset.log 2>&1
```
